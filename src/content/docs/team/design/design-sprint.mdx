# Sprint Docs

<div class="bg-[#f6f8fa] border border-[#e5e7eb] rounded-lg p-6 my-8">
  <p class="text-[17px] leading-7 text-[#1a1f36]">
    Design sprints are structured five-day processes for answering critical business questions through design, prototyping, and testing ideas with customers.
  </p>
</div>

## Sprint Structure

### Day 1: Map
- Define long-term goal
- List sprint questions
- Create process map
- Interview experts
- Choose target

### Day 2: Sketch
- Lightning demos
- Note taking
- Idea generation
- Crazy 8s
- Solution sketch

### Day 3: Decide
- Sticky decision
- Heat map
- Speed critique
- Straw poll
- Supervote

### Day 4: Prototype
- Create storyboard
- Assign roles
- Design prototype
- Tools & assets
- Trial run

### Day 5: Test
- Interview customers
- Observe reactions
- Learn & iterate
- Document findings
- Plan next steps

## Templates

### Sprint Brief
- Problem statement
- Success metrics
- Target audience
- Key stakeholders
- Timeline & logistics

### Daily Worksheets
- Activities checklist
- Time allocations
- Required materials
- Output formats
- Team roles

### Documentation
- Sprint notes
- Decision points
- Prototype details
- User feedback
- Action items

## Best Practices

1. Preparation
- Clear objectives
- Right team members
- Dedicated space
- Required materials
- No distractions

2. Facilitation
- Time management
- Equal participation
- Clear decisions
- Documentation
- Energy management

3. Follow-up
- Summarize findings
- Share learnings
- Plan implementation
- Schedule check-ins
- Measure impact
